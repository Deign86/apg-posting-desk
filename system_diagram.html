Alpha Premiere Group

Posting Desk System Flow

Human-in-the-loop PWA • Firebase • MCP • NVIDIA NIM

Operator + Frontend

Firebase Google Sign-In

Operator signs in from the Vite PWA dashboard. The Firebase Google provider uses the hosted-domain parameter so only APG Workspace accounts should enter live mode.

↓

Process Next Property

The dashboard can pull the next assignment from Firestore property_queue, so interns do not need to type every property name manually.

↓

Workflow Progress Guide

The UI locks steps in order:

  1. Download All Images
  2. Copy Caption
  3. Open Facebook Profile/Alt Account
  4. Paste live Facebook URL
  5. Click to Log Post

Backend Orchestration

FastAPI Auth Middleware

The frontend sends Authorization: Bearer Firebase ID token. Firebase Admin verifies it before live prepare, queue, or log routes run.

↓

Route A: Prepare Assets

POST /api/prepare receives a property name, then coordinates Google Workspace MCP and NVIDIA NIM.

↓

google-docs-mcp Fetch Phase

Drive MCP tools locate the APG Listing Drive folder and fetch content:

  • search_files finds the property folder and files.
  • download_file_content retrieves 3+ images.
  • readDocument extracts caption deets text.
↓

NVIDIA NIM Caption Generation

The backend uses the standard OpenAI client pointed at https://integrate.api.nvidia.com/v1 with stepfun-ai/step-3.7-flash.

Prompt rules: no emojis, no "least term", no "negotiables".

↓

Human Checkpoint

The PWA displays generated caption, images, zip download, and manual instructions. The operator posts through Facebook Web. The system does not use Facebook Graph API.

↓

Route B: Log Manual Post

POST /api/log accepts the live Facebook URL pasted by the operator.

External Systems + Data

Firebase Auth

Google provider login, APG hosted-domain restriction, and Firebase Admin token verification.

Firestore Queue

property_queue stores Ma'am Jean's assigned properties with pending, processing, and completed states.

APG Listing Drive

Property folders contain images and caption deets documents accessed through google-docs-mcp tools.

Facebook Web

Operator manually creates the post using downloaded assets and copied caption, then returns the live URL.

Posting Tracker + Daily Report

google-docs-mcp logging phase uses appendRows for Sheets and appendText for Docs.

Frontend

Vite PWA, Firebase JS SDK, installable manifest, service worker, and gated workflow checklist for interns/operators.

Backend

FastAPI service with Firebase Admin auth, Firestore queue claims, review pipeline, and NIM caption generation.

Workspace MCP

a-bonus/google-docs-mcp handles Drive, Docs, and Sheets operations instead of custom Google API wrappers.

Current design is semi-automated by intent: the system prepares assets and logs results, while the operator performs the actual Facebook publish step.
